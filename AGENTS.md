This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

## Talking with the person who uses this project

- They are **not** a professional coder. Do the technical work yourself; don’t expect them to already know how tools work.
- Use **plain, everyday English**. Short sentences. No slang like “ship it” or “LGTM” unless they use it first.
- **Avoid insider words** (terminal, npm, localhost, repo, API, deploy, branch, merge, etc.). If you must use one, say what it means in simple words the first time—like you’re explaining to a smart friend, not a computer science class.
- Prefer **steps they can picture**: “Open this app (Cursor). Click File, then Open Folder, pick your project folder” instead of only naming tools they may never have heard of.
- If something could be done **inside Cursor** or **in the normal web browser**, say it that way instead of only using product jargon.
